Friends Noonsong – Technical Write-up

Unity Client | Team Project

This page provides a deeper explanation of the key technical challenges I handled in the project, focusing on system integration and problem-solving.

1. Integrating GPS-based Spawn System with AR Collection

Background

The project already had a GPS-based spawn system implemented by a teammate. Objects were spawned based on building areas, while AR collection and the collection system were developed separately. Integrating these independent systems into a single gameplay flow was the main challenge.

Understanding the Existing Spawn Logic

Buildings were divided into rectangular zones. When the player entered a zone, the corresponding spawn script became active and spawned objects specific to that building. Each character was defined as a ScriptableObject with metadata, including its spawn location.

My Role

I analyzed the existing spawn logic and added interfaces needed for integration:

  • Added a function to return the current building name when entering a zone
  • Implemented filtering logic to retrieve spawnable characters based on building name
  • Connected spawn data with AR collection and the collection system

Core Problem

Although objects were spawning correctly, multiple issues appeared during AR interaction:

  • Objects visible on screen could not be collected
  • Collected objects were added to the collection but remained in the world
  • Duplicate spawns occurred due to timing mismatches
  • Incorrect scale or rotation in AR view

Solution

To stabilize the integration, I focused on unifying interaction and state handling:

  • Introduced screen-visibility-based targeting for AR collection
  • Mapped spawned GameObjects directly to their data entries
  • Standardized the collection flow (collect → register → remove → update UI)
  • Added destruction checks to prevent duplicate spawns

Result

The spawn, AR collection, and collection systems were successfully unified into a single, stable gameplay flow. This significantly improved AR interaction reliability and reduced state-related bugs.

2. Stabilizing AR Collection After Beta Test

Issues Found During Beta

After the beta release, several issues were reported during real-world usage:

  • Objects appeared on screen but were not collectible
  • Duplicate objects appeared after respawn cycles
  • Collection state and world state became inconsistent

Root Cause

The core issue was that multiple systems managed object state independently, which caused inconsistencies in AR environments.

Fixes Applied

  • Unified collection target management
  • Fixed the order of collection processing
  • Improved spawn cleanup logic during respawn intervals

Outcome

AR collection became more stable, and duplicate or inconsistent states were eliminated. The fixes significantly improved the overall user experience during gameplay.
